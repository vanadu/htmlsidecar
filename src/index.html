DimWHIT is a keyboard-centric web app for working with HTML email images. With DimWHIT and a few keystrokes, you can gather all the image dimensioning data you need for a specific image and table width and copy your <img> tag and media queries to the clipboard with just a few keystrokes.

Working with DimWHIT is self-explanatory. Just drop the image in the drop area and get to work. If you need any tips on getting the most out of DimWHIT, click here.

!tl;dr quickstart

DIMWHIT is:

  • A compact web app that runs in any modern browser and gives coders all the image data they need to create HTML emails quickly and efficiently.
  • A one-click tool for outputting fully dimensioned <img>, <td>; and <table> code blocks into a code document.

The best way to learn what DIMWHIT can do for you is to just drop some images onto the image viewer area, click the Clipboard button ( ), add a class name, play around with the selections, paste the Clipboard into your text editor and have a look-see. Then come back and read the tl;dr description below for details.

!tl;dr DIMWHIT User's Guide

Dimension Viewers

When you drop an image in the drop area, DIMWHIT displays dimension data in the Dimension Viewers.

Display Resize Tools

You can change the display size of the image using the Display Resize Tools on the DIMWHIT toolbar. Use the buttons to increase the display width of the image by the increment shown. Use the W (px) and H (px) fields to enter custom pixel width and height values as positive integers (px units are added automatically).

DIMWHIT resizes the image, keeping the same aspect ratio, and updates the Display Size values as well as any applicable alerts.

Viewer Width Field

The Viewer is the area with the white email icon pattern on the background. It corresponds to the size of the images’ parent table in the Clipboard output. You set the width of the Viewer with the Viewer Width field.

The default value in the Viewer Width field is 650, which is a common max-width for the content area of HTML emails. You can set it to any table width up to 800 or smaller, depending on whether you want to create a full-width or narrower table, i.e. for a multi-column layout.

To change the viewer width, enter a value in the Viewer Width field. If the value is greater than the current images' display width, DIMWHIT resizes the Viewer up to the maximum app width of 800px and updates applicable alerts. If you enter a Viewer width that is smaller than the current image display width, DIMWHIT flashes an error. To set a viewer width that is smaller than the current images' display width, first change the display width of the image, then change the viewer width accordingly.

Clipboard Functions

When you click on the Clipboard button, DIMWHIT shows you options for outputting an <img>, <td>, <table> tag and/or img.class definitions to the Clipboard so you can paste them into your code document.
